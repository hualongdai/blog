## 说在前面的话

#### 为啥要试用 AMD/CMD？

1. 避免命名冲突
2. 避免污染全局变量


#### 不使用模块化？
可以 ，咱可以使用闭包

比如：

```js
// A
(function () {
	var a = '111';
	console.log(a)
})();

// B
(function () {
	var a = '222';
	console.log(a)
})();

```

假设 B 要用 A 的值 

```js
// A
var modA = (function () {
	var a = '111';
	return a;
})();

// B
var modB = (function () {
	console.log(modA);
})();

```


### 这样做依然会造成全局变量污染，于是 我们引入了 AMD/CMD

### 于是我们引入 AMD/CMD

```js
define('mod1',function(){
	return 'this is mod one'
});


// AMD
define('mod2',['mod1'],function(){
	console.log(mod2)
});

// CMD
define(function(require,exports,module){
	var _mod1 = require('mod1');
	console.log(_mod1)
})

```
